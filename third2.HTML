Question About Git

  1. What is git? What are the Advantages of using git?
  2. What do you understand by the term 'Version control System'?
  3. What's the difference between git and github?

The answers to the three questions are given below.

1. Ans the First question

  1. Git is a distributed version control system(DVCS) that is used to track changes in source code or projects.
    • Advantages of Git:
      1. We can save all versions of the code.
      2. Many developers can work simultaneously.
      3. It is possible to revert to any previous version.
      4. Work can be done offline.
      5. It is fast and secure.
      6. Branching and merging are very easy.

      Branching and merging are explained here

      • BRANCH: A branch represents an independent line of development.

        • Example: Suppose you have a main project. If you want to develop a new feature but donâ€™t want to affect the main code, you create a branch.

      1. To create a new feature
      2. To correct a bug
      3. To protect the main code.
      4. Multiple developers can collaborate at the same time.

      • MERGE: Merge means combining the work of one branch with another branch.

        • Example: Suppose you have finished the work on a branch, and now you want to bring it into the main branch, you perform a merge.

      1. To add a new feature to the main project.
      2. To bring a bug fix into the main branch.

2. Ans the second question

  • A version control system is a system that keeps a history of changes made to any file or source code.

    • The main purposes of a version control system are given below.

    1. Tracking who made what changes and when.
    2. Restoring previous versions.
    3. Making team collaboration easier.[Example: GIT, SVN, MERCURIAL]

3. Ans the third question

  • The difference between Git and GitHub can be explained clearly like this:
  1. GIT:
  • Git is a version control system (VCS).
  • It is used to track changes in source code and manage different versions of a project.
  • Git is installed locally on your computer and works offline.
  • Example: You can create commits, branches, and merge changes on your local machine using Git.
  1. GITHUB
  • GitHub is a cloud-based platform for hosting Git repositories.
  • It allows collaboration by sharing your code with others online.
  • GitHub provides features like pull requests, issue tracking, and project management.
  • Example: You push your local Git repository to GitHub to share it with a team.
